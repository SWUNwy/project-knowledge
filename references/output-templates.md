```markdown
{实体列表}

- 变量命名：{camelCase / snake_case}
- 数据库字段：{snake_case}
- API 字段：{camelCase}

### 关键发现
- {发现1}
- {发现2}
- {发现3}

### 实现程度速览
| 维度 | 状态 |
|------|------|
| 核心管线 | {Demo 完成 / Live 打通 / 等} |
| 类型检查 | {通过 / 未配置} |
| 测试 | {有 / 无} |
| 已知限制 | {主要限制概括} |

> 置信度：{高/中/低}（基于{全量/抽样/概览}分析）
> 分析时间：{N} 次文件读取

准备生成以下文件到 `.claude/knowledge/`：
- INDEX.md — 知识库索引
- points.md — 知识点库（预计 {N} 条）
- term-mapping.md — 术语映射表
- kbase/architecture.md — 架构知识
- kbase/api-design.md — API 设计知识
- kbase/database.md — 数据库知识
- kbase/frontend.md — 前端知识（如适用）
- kbase/workflow.md — 开发工作流（如适用）

是否生成？(y/n)
```

用户确认后执行以下模板生成。**每个文件生成后立即执行该文件对应的 DoD 检查**，通过后再生成下一个文件。

```
## 双路召回机制

### 1. KBase 语义召回
- **方式**：向量检索找「语义相关」的内容
- **适用**：模糊需求场景
- **特点**：覆盖面广，可能包含噪音

### 2. 索引导航式召回
- **方式**：先筛选文档 → 再精确搜索
- **适用**：精确定位场景
- **特点**：可追溯、可解释
```

## points.md 模板

细粒度知识单元库。从 L3 模式采样和 L4 实体提取中提取知识点。

**知识点分类**：

| 分类 | 编号 | 来源 |
|------|------|------|
| 项目定位 | KP-001 ~ KP-003 | L1 配置 + README：项目目标、目标用户、核心交互 |
| 代码定位 | KP-004 ~ KP-012 | L2 结构扫描：核心文件位置 |
| 业务术语 | KP-013 ~ KP-022 | L4 实体提取：领域概念 |
| 技术决策 | KP-023 ~ KP-032 | L1 配置读取：技术选型理由 |
| 实现程度 | KP-033 ~ KP-036 | L3 模式采样 + 测试/构建脚本：各模块完成状态、已知限制 |
| 常见陷阱 | KP-037 ~ KP-046 | L3 模式采样：已知坑和注意事项 |
| 项目规范 | KP-047 ~ KP-056 | L3 模式采样：Git 规范、编码规范 |

**知识点模板**：

```markdown
### KP-NNN: {知识点标题}
| 字段 | 内容 |
|------|------|
| **触发关键词** | keyword1, keyword2, ... |
| **知识点** | {核心知识描述} |
| **相关文件** | {文件路径} |
```

**知识点索引表**：每添加一个知识点，在文件末尾的索引表中追加：

```markdown
| KP-NNN | {类型} | {关键词} |
```

**知识点质量规则**：
- 唯一性：每个知识点覆盖一个独立概念
- 可触发：触发关键词必须是实际会被需求描述使用的自然语言
- 精准定位：必须包含准确的代码路径，而非模糊描述
- 可操作：读者看到后应能直接采取行动

## term-mapping.md 模板

术语映射表，解决业务 ≠ 代码 ≠ API 的命名脱节问题。

**实体术语映射**（从 L4 实体提取生成）：

```
| 业务术语 | 英文术语 | 代码命名 | API 路径 | 数据库表 |
|---------|---------|---------|---------|---------|
| {术语} | {Term} | `{Pascal}` / `{camel}` | `/api/{path}` | `{table}` |
```

**字段术语映射**（从 L3 命名采样生成）：

```
| 业务术语 | 英文术语 | 数据库字段 | API 字段 | TypeScript |
|---------|---------|-----------|---------|------------|
| {术语} | {Name} | `{snake}` | `{camel}` | `{camel}: {type}` |
```

**状态术语映射**（如果发现枚举状态）：

```
| 业务术语 | 英文术语 | 数据库值 | 说明 |
|---------|---------|---------|------|
| {术语} | {Term} | {value} | {说明} |
```

**易混淆术语**（如果发现常见错误命名）：

```
| ❌ 错误用法 | ✅ 正确用法 | 说明 |
|------------|------------|------|
| {wrong} | {correct} | {说明} |
```

**新术语添加规范**：

```
当引入新业务概念时，按以下步骤添加：
1. 确定业务术语：与用户确认中文名称
2. 确定英文术语：使用单数形式
3. 确定代码命名：遵循 camelCase/PascalCase
4. 确定 API 路径：使用 kebab-case 复数形式
5. 更新本文件：添加到对应映射表
```

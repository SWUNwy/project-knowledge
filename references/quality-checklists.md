# Quality Checklists

> 每个文件生成后立即执行对应 DoD 检查，不满足不推进。

---

## INDEX.md DoD
- [ ] 双路召回机制描述完整
- [ ] 知识库目录与实际分析结果一致（不包含无数据的分类）
- [ ] 关键词触发表已填充，关键词为实际会被使用的自然语言
- [ ] 模板中的 `{占位符}` 已全部替换为实际内容

## points.md DoD
- [ ] 至少 3 个知识点
- [ ] 每个知识点包含触发关键词、核心知识、相关文件
- [ ] 知识点覆盖至少 2 个分类
- [ ] 知识点质量规则已遵循（唯一性/可触发/精准定位/可操作）

## term-mapping.md DoD
- [ ] 至少包含实体映射表
- [ ] 业务术语 ↔ 代码命名 ↔ API 路径 双向可追溯
- [ ] 如有枚举状态，已包含状态术语映射表

## kbase/ 各文件 DoD
- [ ] 占位符已全部替换为实际内容
- [ ] 内容基于实际分析结果，而非推断
- [ ] 仅当项目类型适用时生成（如 frontend.md 仅对 Web 项目）
- [ ] 每个文件有至少一个实质性的表格或内容块
- [ ] architecture.md 包含项目定位（第零章）和实现程度（最后一章）
- [ ] 实现程度章节包含：总览表、各模块详情、已知限制

## 全局 DoD（所有文件生成完成后）
- [ ] 所有文件的 `*最后更新*` 时间戳使用当日日期
- [ ] 生成的文件数与确认时承诺的一致
- [ ] 无空文件（如某维度无数据，跳过该文件）
- [ ] 不覆盖用户已有手动修改的内容

## P1 门禁检查清单（Step 4→5）

分层扫描完成后，生成前确认：

- [ ] L1 配置已读取并提取关键信息
- [ ] L2 结构已扫描，目录树已生成
- [ ] L3 模式采样已完成，命名/错误处理等模式已识别
- [ ] L4 实体已提取（实体列表、路由、数据库等）
- [ ] 项目类型已确定（来自项目分类器）
- [ ] 分析摘要已向用户展示
- [ ] 用户已确认生成

文件生成后，写入缓存前：

- [ ] INDEX.md DoD 已通过
- [ ] points.md DoD 已通过
- [ ] term-mapping.md DoD 已通过
- [ ] kbase/ 各文件 DoD 已通过
- [ ] 全局 DoD 已通过

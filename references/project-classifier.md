# Project Classifier

> 分析正式开始前先判定项目类型，不同类型影响后续维度的权重和深度。

---

## 判定依据（按优先级）

| 信号文件 | 判定规则 |
|---------|---------|
| `package.json` | 有 `next` → Next.js 全栈；有 `react` → React SPA；仅 CLI 工具 → Node CLI |
| `pom.xml` / `build.gradle` | Java / Kotlin 项目 |
| `go.mod` | Go 项目 |
| `Cargo.toml` | Rust 项目 |
| `Gemfile` | Ruby / Rails 项目 |
| `requirements.txt` / `pyproject.toml` | Python 项目 |
| `Podfile` / `*.xcworkspace` | iOS / macOS 项目 |
| `pubspec.yaml` | Flutter / Dart 项目 |
| 多个语言的项目 | 按 `src/` 或 `lib/` 内容判定主要语言 |
| 以上都没有 | 通用项目（通用维度的子集） |
